    Guessing Game # 

    let foo = 5;	// immutable
let mut bar = 5;	// mutable
let baz = String::new();

    기본적으로 let과 let mut은 변수의 가변성에 차이를 두고 다르게 선언하는 것을 의미.String::new는 String타입의 연관함수로 OO에서 클래스의 인스턴스 없이 호출할 수 있는 정적 메소드와 동일.

    io::stdin()
	.read_line(&mut guess)
	.expect("Failed to read line");

    프로그램의 시작점에 ‘use std::io’로 명시되었기 때문에 함수 호출시의 io::stdin은 표준 입력 핸들 타입인 std::io::Stdin을 돌려줌. &mut guess를 넘겨받아 사용자의 입력을 저장하게 됨. 이때 사용자의 입력은 지속적으로 추가됨으로 가변 문자열이어야 함. 이때 ‘&mut’ 인 이유는 Rust에서는 ‘&’ c참조자의 기본 속성이 안전성을 위해 Immutable임으로 명시적으로 가변임을 지정할 필요가 있음. ‘.expect’ 는 에러를 처리하는 메소드로 인스터스가 Error를 낼 경우 해당 메세지를 출력하며 동작을 멈추게 됨. 만일 ‘expec’없이 컴파일 한다면 경고가 표시됨.

    // place holder
println!("You guessed: {}", guess);
// place holder 지정
println!("You guessed: {g} number {n}", g= guess,n = number);

    위 문장에서 ‘{}’ 부분은 변경자(Place Holder)로 guess의 표시 위치를 나타냄. 하나 이상의 Place Holder를 아래처럼 지정하여 사용할 수도 있음.

    // Cargo.toml
[dependencies]
rand = "0.4.0"

    Random 처리를 위해 Cargo.toml에 의존성을 추가하여 rand crate를 명시함.

    match guess.cmp(&secret_number) {
            Ordering::Less => println!("Too small!"),
            Ordering::Greater => println!("Too big!"),
            Ordering::Equal => {
                println!("You win!");
                break;
			}

    사용자가 입력한 guess 숫자와 Random으로 생성된 secret_number를 비교하는 메소드. std 표준 라이브러리로부터 std::cmp::Ordering Scope를 가져와서 Ordering 타입을 사용하는 형태.

    let guess: u32 = match guess.trim().parse() {
	Ok(num) => num,
	Err(_) => continue,
};

    입력된 guess의 문자열을 secret_number와 비교하기 위해 숫자로 바꾸는 라인. 기존 guess 변수를 triming된 숫자로 Shadowing하여 재사용하는 방식으로 구현됨. ‘guess.trim()‘을 통해 사용자 입력시 들어온 Enter의 개행문자를 제거하고 ‘parse()‘를 통해 숫자열로 파싱. 이때 u32와 같이 정확한 타입을 명시해야함. 사용자가 입력한 숫자에 특수문자 같은 것이 포함되어 u32로 전환할 수 없을 경우를 대비해서 expect와 같은 별도 처리가 필요할 수도 있음. 여기서는 Error가 들어올경우 continue를 통해 다음 Loop로 다시 시작되도록 처리 됨.

    loop {
...
match guess.cmp(&secret_number) {
		...
		Ordering::Equal => {
			println!("You win!");
			break;
		}
}

    main 프로그램은 Ordering::Equal이 나올때까지 Loop로 들어가 반복되며 수행됨.
